Making a programming language

This article discusses the motivation I have for making a new programming language, which I am calling Goat. If you're looking to find out about the language itself, some good places to start are the beginner's introduction at Learn, or the Examples, or how it compares to other languages at Compare.

DISCLAIMER: Creating a useful programming language is a big task. Currently, my language is significantly less useful than the most commonly used ones. If you want to get stuff done, you should look at those languages instead.

I think programming is very interesting. It is also very complicated though. Why that is? Is programming complicated due to the languages we use? Can we make it easier? Since programming languages are the "user interface" for programming, it seems a reasonable place to start investigating. I made a programming language Goat to experiment with some ideas I have for making programming and programs easier to learn and to write. In summary, some key ideas for Goat are:

  • Easier to learn for programming means a small core of concepts that interact in well-defined ways.
  • Easier to learn for programs means readable code with precise meaning.
  • Easier to write for programming means simple, flexible syntax, and the ability to incrementally and interactively develop and combine ideas.
  • Easier to write for programs means the ability to concisely express complex behaviour, and reliable foundations to build on.

Goat is still a work in progress and it remains to be seen whether it will live up to these lofty ideals. :P
