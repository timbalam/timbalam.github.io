Making a programming language

This article discusses the motivation I have for making a new programming language, which I am calling Goat. If you're looking to find out about the language itself, some good places to start are the beginner's introduction at Learn, or the Examples, or how it compares to other languages at Compare.

Languages are about communication. For programming, this communication needs to be precise enough for the computer to follow, and also readable enough for humans to make additions and fix errors. Code is the written form of a programming language.

Creating a useful programming language is a big task. Currently, my language is significantly less useful than the most commonly used ones. If you want to get stuff done, you should look at those languages instead. However, programming languages are fascinating to study in their own right.

Along the same lines as human languages, a programming language provides a set of primitive constructs and combining forms. These forms are ways to put simple parts together to develop more complex behaviour. The choices of primitives and combining forms of a programming language encourages a philosophical approach to writing programs.

One of the philosophical ideas guiding the design of Goat is that fewer concepts mean less to learn for a beginner and a uniform way to approach problems. However, if the combining forms of a language are too simple, expressing complex ideas can be cumbersome. Goat aims for a balance between these concerns with powerful universal combining forms which can cover a wide variety of use cases. In addition, Goat sets out to help the programmer write reliable code with easily predictable behaviour and without crashes.
