Calculating with Goat

In the previous tutorial we used Goat to print some text on the screen. Cool! In our next exercise we will see that Goat can calculate things too. We will need another Goat pen.

For this exercise we will consider the following example code. 

myNumber = 2;
myMaths = 3 * myNumber;
.inspect = myMaths;

Type out (or copy) the code above into the web interpreter box. Upon navigating away from the box you should see the following printed below the box: 

6.0

How it looks and what it means

Below is a diagram like the one from the other tutorial, showing our program with annotations. This time the changed parts of the code are highlighted. (If you are lost it might help to go back over the other tutorial.) 

1 | myNumber = 2;
    ^^^^^^^^
    1          ^
               2
2 | myMaths = 3 * myNumber;
    ^^^^^^^
    3         ^^^^^^^^^^^^
              4
3 | .inspect = myMaths;
              ^^^^^^^
              5
  1. We might recognise the first line as an assignment, this time with the left-hand side name myNumber.
  2. The assignment right-hand side is the literal number 2. Recall that this assignment means that we are choosing the name myNumber to be equivalent to the number 2 where ever we use it in definition code.
  3. The second line is also an assignment, this time to the label myMaths.

  4. The right hand side is a mathematical expression. The asterisk symbol * indicates a multiplication operation applied to the left and right inputs. Here, we are multiplying the number 3 on the left by the value represented by the name myNumber, defined on the previous line.

  5. The third line is where we define the magical inspect name, allowing the Goat web interpreter to evaluate and print it. The definition we have given is the name myMaths, which as we have seen turns out to evaluate to the number 6.

A note on names

The programs we have written so far have involved naming pieces of code via a series of ; separated assignments. In Goat, the assigned names from this series make up a set. Once a name is added to the set its assigned definition cannot change, and it is an error to reuse a name. We can demonstrate this by modifying our program above, to add a duplicate definition for myMaths. The following program has an additional line added compared to the one we made above. 

myNumber = 2;
myMaths = 3 * myNumber;
myMaths = 5;
.inspect = myMaths;
Update the code block we typed earlier in the Goat pen above, and navigate away as usual to run it. You should see an error like the following. 
error: multiple assignments: "myMaths"
The error states that the meaning of myMaths is ambiguous â€“ which value would we expect to be printed? Goat enforces the use of unique names with unchanging values so that it is easier to know what a label means, allowing programs to be more easily understood.

Aside: Since names are required to be unique, we have the property that the order in which we make our assignments doesn't affect the program meaning! Go ahead and try moving the lines of code around in any of the tutorial exercises so far! You'll find the results won't change.
