Run a simple Goat program

The text in the box below will be evaluated by the Goat web interpreter.

We will start with a simple program to test that it is working, and to see how to use it to run Goat programs. In the box above type in the following code block. Click outside the box or press the tab key when you are finished. 

myText = "Hello world.";
.output = myText;

If it worked correctly, you should see the following text appear below the box: 

Hello world.

Congratulations, you have a working Goat program!

How it looks and what it means

In this part we will deconstruct the simple program we just wrote. This is a bit of a look at the program from the computer's perspective (technically the perspective of the Goat interpreter, which is a computer program). Like several notable human languages, the computer reads from left to right. We begin with the first word of the first line, highlighted below (where I have added a line number marker): 

1 | myText = "Hello world.";
    ^^^^^^
The computer understands sequences of alphabetic characters such as myText to be a name. Next is a space. Spaces can increase readability for humans, but they don't matter to the computer! An exception to this that we should note is that names are currently not allowed to contain spaces. After the space is an equals sign symbol =: 
1 | myText = "Hello world.";
           ^
The computer sees the = character and understands that we are making an assignment. The name to the left is being defined as equivalent to the code to the right. Next is another (ignored) space, then a sentence in double quotes "Hello world.". 
1 | myText = "Hello world.";
             ^^^^^^^^^^^^^^
Finally, we have a semi-colon symbol ;. 
1 | myText = "Hello world.";
                           ^
The ; character indicates that we have finished with the definition for the assignment. The content of the definition is the double-quoted sentence "Hello world.". This is an example of a primitive form in Goat, for which the Goat interpreter has built-in knowledge.

The second line, shown below, begins with a period symbol .: 

2 | .output = myText;
    ^
followed by the word output: 
2 | .output = myText;
     ^^^^^^
The . as giving special meaning to whichever word follows it. In this case, it allows the Goat web interpreter to read, evaluate, and print the definition we give to the word output. The line continues with =, following the assignment pattern which we saw in the first line. In this case the assigned code is a repetition of myText: 
2 | .output = myText;
              ^^^^^^
The Goat web interpreter is set up to look for the definition of output, and to print the result of evaluating the definition below the input text box. As we assigned the name myText to be the text "Hello world." on the first line, this is what our program prints.

We can expand on the concept of a primitive form a bit. This example features a double-quoted sentence, called "literal text" or just "text". The form uses double-quotes around text to represent exactly the text shown within the quotes. (Note that inside quotes, spaces are significant and not ignored!) Some more examples of a primitive form are literal numbers – represented by numeric characters e.g. 11, 0.45 – and some mathematical operations – represented by infix symbolic characters e.g. +, /. We will see demonstrations of these primitive in the next example.
