Run a simple Goat program

In this first tutorial we will writing a simple program. This will firstly test that the Goat Web Interpreter is working, and secondly demonstrate how to run a Goat program. For this we will need the box below, called the Goat Pen. As we will see, text written in the Goat Pen can be evaluated by the Goat Web Interpreter.

Type the following code block into the Goat pen above. Click outside the pen or press the tab key when you are finished. 

myText = "Hello world.";
.output = myText;

If it worked correctly, you should see the following text appear below the box: 

Hello world.

Congratulations, you have a working Goat program!

How it looks and what it means

The diagram below shows an annotated version of the program we just wrote, with the main features of interest to the computer highlighted. (Line numbers for the original program lines are shown on the far left.) 

1 | myText = "Hello world.";
    ^^^^^^   ^^^^^^^^^^^^^^
    1      ^ 3             ^
           2               4
2 | .output = myText;
    ^       ^       ^
    5^^^^^^ 2 ^^^^^^4
     6        7

  1. The computer understands sequences of alphabetic characters such as myText to be a name. The space at the end helps readability but is not necessary and doesn't change the meaning of the program. Names are not allowed to contain spaces.

  2. The computer sees the equals sign symbol = and understands that we are making an assignment. The name to the left is defined as equivalent to the code to the right. Any space immediately after the = also has no effect on the program.

  3. The sentence in double quotes "Hello world." represents a literal text value.

  4. Each line ends with a semi-colon symbol ;. The ; indicates that we have finished with the definition for an assignment.

  5. The second line, shown below, begins with a full-stop symbol . . The . indicates that the following name has a special meaning. In this case, it allows the Goat web interpreter to read and evaluate the definition of output.

  6. The Goat pen is set up so that the definition made for output is evaluated and the result printed below the pen.

  7. The name myText is assigned to output. When used inside a definition, a name is a place-holder for its definition. As myText was assigned to be the text "Hello world.", this is what output evaluates to, and is thus printed below the Goat pen.

My first error

It is an error to use a label in a definition without giving a definition too. Let's see what happens if we change the definition of output to an undefined label myTekts. Go to the Goat pen above and change the second line to the following: 

.output = myTekts;
