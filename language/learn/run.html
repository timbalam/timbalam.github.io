Run a simple Goat program

In this first tutorial we will write and run a simple program.

We will use the box below, called the Goat Pen. As we will see, the Goat Pen can evaluate Goat code written in it. 





The box following contains some Goat code.
Type the code into the Goat pen above.
Click outside the pen or press the tab key when you are finished.




message = "Hello darkness.";
.inspect = message;





If it worked correctly, you should see the following text appear below the box:




Something's not working...





Congratulations,
you have made a working Goat program!




How it looks and what it means




A Goat program structure is inspired by that of a dictionary.
In the simplest form,
a program describes a vocabulary â€”
a series of words and their corresponding definitions.





Our program defines two words,
with the form word = definition ;.
On the first line,
message is defined as the piece of text "Hello, darkness.".





On the second line,
inspect is defined.
The word inspect
is written with a prefix period character ..
We will see the significance of this below and in later tutorials.





inspect is defined effectively as a synonym for message,
demonstrating that Goat,
like a dictionary,
allows words to be defined in terms of other words. 





We make this synonym for a good reason,
because inspect is a privledged word inside the Goat Pen.
When made with the . prefix,
the word inspect defines the value that is displayed in the output box.





Since inspect is a synonym for message,
and message is a piece of text,
we end up seeing that text displayed:
"Hello, darkness.".




New syntax




In this tutorial we have encountered our first example of Goat code.
Let's summarise:




    

  • 
A program is a series of word-definition pairs.

  • 
An equals sign = is used between a word being defined and its definition.
The end of the definition is indicated by a semi-colon character ;.

  • 
A word can contain letters and numbers and must start with a letter.
Spaces or symbols are not allowed in words.

  • 
A fragment of literal text can be represented by surrounding it with double quote characters ".

Below is a list of additional facts and exercises for those who wish to learn more about Goat syntax.
For the exercises, utilise the Goat Pen at the top of the article. 
(Remember to press tab or click outside of the box after making the changes).




    

  • 
The example code in this tutorial uses white space (spaces and newlines) to improve readability.
In fact, space can be added almost anywhere without changing the program behaviour.
Delete all of the spaces and newlines in the example exercise.

  • 
Next, add back in any spaces or newlines you like.
Can you produce an error?

  • 
One place that spaces are forbidden is in words.
Add a space in message.
(This should give an error!)

  • 
The definition of message in line 1 uses a literal text.
Change the text inside the "'s.

    • 

  • 
The definition of inspect makes reference to another defined word message.
Modify the definition of inspect to be an undefined word, e.g. messge.
(Should give another error!)
Note that if we misspell or misremember a word we defined,
Goat will let us know so we can fix it.
