Run a simple Goat program

In this first tutorial we will writing a simple program. This will firstly test that the Goat Web Interpreter is working, and secondly demonstrate how to run a Goat program. For this we will need the box below, called the Goat Pen. As we will see, text written in the Goat Pen can be evaluated by the Goat Web Interpreter.

Type the following code block into the Goat pen above. Click outside the pen or press the tab key when you are finished. 

myText = "Hello darkness.";
.inspect = myText;

If it worked correctly, you should see the following text appear below the box: 

Hello darkness.

Congratulations, you have a working Goat program!

How it looks and what it means

The diagram below shows an annotated version of the program we just wrote, with the main features of interest to the computer highlighted. (Line numbers for the original program lines are shown on the far left.) 

1 | myText = "Hello darkness.";
    ^^^^^^   ^^^^^^^^^^^^^^^^^
    1      ^ 3                ^
           2                  4
2 | .inspect = myText;
    ^^^^^^^^   ^^^^^^
    5        ^ 6     ^
             2       4
  1. The computer understands sequences of alphabetic characters such as myText to be a name. The space after isn't necessary but helps readability. For the most part, spacing doesn't affect the meaning of the program. Names are not allowed to contain spaces.
  2. The computer sees the equals sign symbol = and understands that we are making an assignment. The name to the left is defined as equivalent to the code to the right. (Any space immediately after the = also has no effect on the program.)
  3. The sentence in double quotes "Hello darkness." represents a literal text value.
  4. Each line ends with a semi-colon symbol ;. The ; indicates that we have finished with the definition for an assignment.
  5. The second line defines a special name inspect with a full stop symbol . prefix. If this name is defined (the prefix is necessary), the Goat Pen will try to evaluate the definition and prints the result below the pen. Otherwise, a default message will be displayed.
  6. The name myText is assigned to inspect. When used inside a definition, a name is a place-holder for its definition. As myText was assigned to be the text "Hello darkness.", this is what inspect evaluates to, and is thus printed below the Goat pen.

My first error

It is an error to use a label in a definition without giving a definition too. Let's see what happens if we change the definition of inspect to an undefined label myTekts. Go to the Goat pen above and change the second line to the following: 

.inspect = myTekts;
You should see the following message following the Goat pen: 
<error: Missing assignment: myTekts>
This way, if we might misspell or misremember a name, Goat will let us know about it so we can fix it!
