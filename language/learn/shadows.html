Regarding precedence

In the previous tutorial we mentioned how the block declaration introduces a nested layer of names, permitted to overlap with outer layers. In this tutorial we will investigate how layers of names created by blocks interact with outer layer of names. For this we will need our Goat Pen:

Let's modify the example from the other tutorial, and instead define x locally, above the first line, as shown below. 

x = 1;
myLine = { .y = x + 2 };
.inspect = myLine.y;

Typing this into the Goat Pen and running should get us the following result: 

3.0

For the second part of our investigation, we are going to add back the x definition to myLine. With this change, our program becomes: 

x = 1;
myLine = { .x = 2; .y = x + 2 };
.inspect = myLine.y;

What will the value of myLine.y turn out to be? Type the new program into the Goat Pen to find out. You should see this. 

4.0

This result is the same as the program from other tutorial, and demonstrates how Goat decides what definitions to give to names. The definition for "x" made inside the block definition overrides the definition made on the first line of the program. For Goat program the following holds: Names assigned in a block take precedence over (or "shadow") any identical names defined in layers outside the block. The value represented by a name used in a definition hence comes from the closest surrounding layer where it is given a definition.
